Cette année, nous allons attaquer la partie dynamique d'un site web. Cela signifie qu'une base de données est ajouté à votre site permettant un système d'utilisateurs qui auront chacun leur biens.

IMPORTANT : à la fin de chaque étape, push ton projet sur Github

  1. Étape 1

    Constuire des formulaires

    1. Transformer tous ses fichiers html en fichiers php
    2. Créer d'un formulaire avec des fieldset, legend, des input type text et ses labels
    3. Ajouter des inputs d'autres types (number, password, email, date, checkbox, ...)
    4. Ajouter du bouton submit pour envoyer les données
    5. Mettre les attributs telles que palceholder, min, max, value, ...
  2. Étape 2

    Ajouter la base de données

    1. Rappel de l'UAA7 de la 4ème
    2. Préparer le serveur hebergeant la base de données
    3. Créer la base de données ainsi que les différentes entités liées à son MLD
    4. Ajouter des données dans la base permettant de l'utiliser
    5. Connecter son site à la base de données
    6. Faire une première requête propre avec le prepare, execute et fetchAll pour récupérer les biens
  3. Étape 3

    Structurer son code avec MVC

    1. Comprendre le système MVC pour mieux structurer son projet
    2. Adapter son arborescence pour correspondre au modèle MVC
    3. Créer le routage dans le controller pour passer d'une page à l'autre selon l'URL
    4. Afficher la première page avec tous les biens
    5. Création d'un premier component navbar
    6. Afficher les pages connexion et inscription
  4. Étape 4

    CRUD utilisateur

    1. Traiter le formulaire d'inscription avec $_POST
    2. Vérifier que les données ne sont pas vide côté serveur
    3. Vérifier l'unicité des données selon des critères pertinents
    4. Gérer la variable globale $_SESSION
    5. Traiter la connexion/déconnexion de l'utilisateur avec le fetch
    6. Changer l'affichage du menu selon l'état de l'utilisateur
    7. Permettre à l'utilisateur de modifier son profil
    8. Permettre à l'utilisateur de supprimer son profil
  5. Étape 5

    CRUD biens

    1. Afficher la page des biens personnels
    2. Permettre à l'utilisateur de modifier son bien
    3. Permettre à l'utilisateur de supprimer son bien
    4. Permettre à l'utilisateur de créer un nouveau bien
  6. Étape 6

    Partie sécurité

    1. Éviter les injections SQL depuis un formulaire
    2. Empêcher une personne de supprimer/modifier le bien de quelqu'un d'autre
    3. Éviter les attaques XSS
  7. Étape 7

    Bonus

    1. Gérer les images dans la base de données
    2. Ajouter des flash messages avec la bonne couleur
    3. Héberger son site sur heroku
    4. Ajouter des rôles aux utilisateurs de son site
    5. Ajouter un système de filtrage sur les biens avec $_GET
